# Execution Plans

실행 계획은 일급 아티팩트다. 복잡한 작업은 반드시 실행 계획을 먼저 작성한다.

## 디렉터리 구조

```
exec-plans/
├── active/       ← 진행 중인 계획
├── completed/    ← 완료된 계획 (의사결정 로그 보존)
└── README.md     ← 이 파일
```

## 실행 계획 템플릿

```markdown
# [계획 제목]

## 목표
한 문장으로 요약.

## 배경
왜 이 작업이 필요한가.

## 단계
1. [ ] 단계 1 — 설명
2. [ ] 단계 2 — 설명
3. [ ] 단계 3 — 설명

## 영향 범위
- 수정 대상 모듈/패키지
- 관련 테스트

## 의사결정 로그
| 날짜 | 결정 | 이유 |
|---|---|---|
| YYYY-MM-DD | 선택한 접근 | 왜 이 접근을 선택했는가 |

## 완료 조건
- [ ] 모든 테스트 통과
- [ ] 아키텍처 테스트 통과
- [ ] 관련 문서 업데이트
```

## 규칙

- 작업 시작 전에 `active/`에 계획 파일을 생성한다
- 진행 중 의사결정은 로그에 기록한다
- 완료 후 `completed/`로 이동한다 (삭제하지 않는다)
- 단순한 버그 수정에는 실행 계획이 필요 없다
